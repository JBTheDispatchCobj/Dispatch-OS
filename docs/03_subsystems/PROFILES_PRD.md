# Profiles PRD

Profiles exist for institutions, people, startups, vendors, products, cartridges, and relationships. Profiles combine public facts, derived metrics, inferences, verified information, private context, goals, behavior, access, and activity.

## Required sections for implementation
- Goals and non-goals
- Object contracts
- Inputs and outputs
- Rules and workflows
- Permissions and visibility
- Model usage and escalation
- Metrics and tests
- Failure modes
