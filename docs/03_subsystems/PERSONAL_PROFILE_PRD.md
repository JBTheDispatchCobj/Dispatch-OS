# Personal Profile PRD

Model the person through role, responsibilities, institution, goals, projects, interests, relationships, decisions, reading behavior, access, and preferred channels. Greater system access improves operational capacity while respecting permissions.

## Required sections for implementation
- Goals and non-goals
- Object contracts
- Inputs and outputs
- Rules and workflows
- Permissions and visibility
- Model usage and escalation
- Metrics and tests
- Failure modes
