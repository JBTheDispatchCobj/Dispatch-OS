# RFC-11003 — Financial Infrastructure and Fintech Capability Model

Status: Draft 1.0

Owner: Auric Works

Authority: Dispatch Constitution and Volumes I–X

## Capability-first model

Every fintech and infrastructure provider is represented through capabilities, products, rails, dependencies, integrations, regulated roles, customers, evidence, and operating maturity.

## Infrastructure domains

Core banking, digital banking, ledger, payments, card issuing, acquiring, processing, ACH, wire, RTP, FedNow, identity, KYC/KYB, AML, fraud, lending, servicing, collections, treasury, data, analytics, cybersecurity, compliance, wealth, insurance, accounting, communications, and AI.

## Rule

Vendor names never substitute for capability definitions.
