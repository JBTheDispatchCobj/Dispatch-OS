# RFC-11000 — Volume XI Architecture & Dependency Map

Status: Draft 1.0

Owner: Auric Works

Authority: Dispatch Constitution and Volumes I–X

## Purpose

Define the domain layer that makes Dispatch an operating system for institutional finance rather than a generic business operating system.

## Dependencies

Kernel → Object Registry → Knowledge Graph → Agent Harness → Execution Engine → Connector Registry → Institutional Intelligence → Cooperative Markets Cartridge → The Auric and Terminal.

## Owned capabilities

Institution ontology, financial infrastructure taxonomy, product taxonomy, cooperative-finance model, innovation lifecycle, institution and fintech profiles, relationship graph, readiness, fit, regulatory applicability, evidence packages, and public/private projections.

## Deliverables

Canonical domain objects, relationship types, truth models, rule families, workflows, agents, profiles, sources, APIs, scorecards, and acceptance tests.
