# FS-10000 Wireframe Descriptions

## Terminal Shell
Persistent left navigation; top context header; primary content canvas; optional right evidence/activity drawer. The context header shows tenant, workspace, institution, role, environment and freshness.

## Home Command Center
Top row: priority alerts and approvals. Middle: active workflows, assigned tasks and top opportunities. Bottom: freshness exceptions, upcoming meetings and recent activity. Every card supports drill-through and permission-aware actions.

## Institution Terminal
Object header followed by summary band and tabbed workspace. Summary band contains identity, financial snapshot, risk, readiness and evidence confidence. Tabs lazy-load independently and retain route state.

## Opportunity Workspace
Left: qualification, economics and readiness. Center: recommendation, evidence and workflow. Right: owner, status, approvals and activity. Decision controls are visually separated from automated recommendations.

## Workflow Workspace
Progress rail across top; step list and dependencies at left; active step center; evidence, approvals and timeline at right. Failed or blocked steps expose reason, owner and safe recovery path.

## Mobile Restricted Mode
Supports alerts, tasks, approvals, meeting briefs and object lookup. High-risk configuration, bulk export and complex graph editing remain desktop-only.
