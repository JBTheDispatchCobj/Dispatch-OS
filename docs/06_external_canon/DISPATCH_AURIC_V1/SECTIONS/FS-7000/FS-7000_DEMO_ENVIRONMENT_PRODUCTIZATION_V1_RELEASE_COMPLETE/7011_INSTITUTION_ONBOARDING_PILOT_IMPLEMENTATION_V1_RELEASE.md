# 7011 — Institution Onboarding & Pilot Implementation V1

## Purpose

Define how a real institution moves from discovery to a controlled Dispatch pilot.

## Pilot Phases

1. Qualification
2. Scope
3. Data and permissions
4. Profile build
5. Workflow selection
6. Configuration
7. User testing
8. Controlled launch
9. Value measurement
10. Scale decision

## Qualification Criteria

- executive sponsor
- defined pain
- available owner
- data path
- acceptable security posture
- realistic workflow
- measurable outcome
- approval path
- budget or sponsored pilot structure

## Discovery Inputs

- institution overview
- organizational chart
- strategic priorities
- vendor/system inventory
- target workflow
- policies/controls
- sample documents/data
- current process map
- users
- desired output
- risk constraints

## Pilot Scope

Must define:

- institution
- workflow
- users
- systems
- data
- documents
- agents
- evidence
- approvals
- outputs
- duration
- success metrics
- exclusions
- exit plan

## Data Access Modes

- manual upload
- scheduled export
- read-only connector
- sandbox
- limited production
- attested data
- public data

## Implementation Workstreams

- governance
- security
- data
- objects/profile
- workflow
- agents
- evidence
- approvals
- UI
- training
- measurement

## First 30 Days

Week 1:
- scope
- owners
- data
- security
- baseline

Week 2:
- institution profile
- workflow mapping
- fixture/test data
- permissions

Week 3:
- user testing
- evidence validation
- dashboard setup
- training

Week 4:
- controlled launch
- monitoring
- executive review
- scale recommendation

## Pilot Outputs

- configured Terminal
- institution profile
- selected workflow
- agent team
- evidence map
- approval map
- dashboard
- user guide
- value report
- scale plan

## Acceptance Tests

- Scope is bounded.
- Data access is approved.
- Human authority remains intact.
- Baseline and outcome metrics exist.
- Pilot can be discontinued cleanly.
- Scale decision is evidence-backed.
