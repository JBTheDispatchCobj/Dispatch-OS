# 7002 — Fintech Pilot Readiness Demo V1

## Purpose

Demonstrate how Dispatch converts a fintech pitch into an institution-ready pilot plan.

## Demo Thesis

The platform should answer:

- What problem does the fintech solve?
- Which institutions have that problem?
- Is the institution operationally ready?
- Can the fintech integrate?
- What data is involved?
- What regulatory and vendor-risk reviews apply?
- Who must approve the pilot?
- What would success look like?
- Could an investment or broader partnership follow?

## Scenario

Use the synthetic credit union from 7001 and a fictional fintech offering AI-enabled member-service workflow automation.

The fintech should have:

- seed or Series A stage
- 15–30 employees
- early revenue
- two financial-institution references
- cloud-hosted platform
- API and file-based integration options
- SOC 2 Type II
- no mature credit-union implementation playbook
- incomplete model-risk documentation
- strong use-case fit
- a realistic pilot cost

## Workflow

1. Ingest startup deck and documents.
2. Create startup profile.
3. Extract capabilities and claims.
4. Match use case to institutional pain.
5. Resolve institution systems and owners.
6. Map integration and data flow.
7. Run vendor-risk checklist.
8. Run compliance applicability review.
9. Score readiness.
10. Identify blockers and conditions.
11. Draft pilot plan.
12. Create approval path.
13. Draft board or executive memo.
14. Track pilot execution.
15. evaluate pilot-to-scale or pilot-to-invest.

- use case

- member/customer impact
- data scope
- systems

- security controls
- compliance controls
- timeline
- training

- success metrics

- exit plan

## Success Metrics

Examples:

- average handle-time reduction
- workflow completion time
- employee adoption
- member satisfaction
- error rate
- escalation rate
- compliance exceptions
- cost per interaction
- data-quality incidents
- support tickets

## Approval Path

Required:

- business owner
- technology owner
- security
- vendor risk
- compliance
- legal if contract or data issues
- executive sponsor
- board if policy or material-risk threshold requires

## Demo Interactions

- Is this fintech ready for this credit union?
- What blocks the pilot?
- Which systems must integrate?
- What data does the vendor receive?
- What evidence is still missing?
- Which executives must approve?
- Draft the pilot plan.
- Draft the board brief.
- Compare this fintech with two alternatives.
- What would justify a strategic investment?

## Acceptance Tests

- Startup marketing claims are separated from verified evidence.
- Readiness scores explain blockers.
- Security and compliance gates cannot be bypassed.
- Pilot outputs use institution-specific systems and owners.
- Investment is treated as optional and downstream.
- All external commitments require human approval.
