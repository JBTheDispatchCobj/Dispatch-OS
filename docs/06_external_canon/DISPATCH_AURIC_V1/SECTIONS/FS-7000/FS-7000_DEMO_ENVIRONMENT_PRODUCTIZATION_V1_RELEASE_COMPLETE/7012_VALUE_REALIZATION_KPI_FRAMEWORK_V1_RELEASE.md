# 7012 — Value Realization & KPI Framework V1

## Purpose

Measure whether Dispatch creates institutional value beyond novelty.

## Value Categories

- revenue
- fee income
- spread
- cost savings
- loss avoidance
- compliance efficiency
- reduced cycle time
- improved evidence quality
- increased readiness
- improved decision speed
- user adoption
- member/customer experience
- strategic option value

## Baseline Object

- metric
- current value
- period
- source
- owner
- confidence
- collection method

## Outcome Object

- metric
- target
- actual
- variance
- period
- source
- attribution
- confidence
- approval

## Core Platform KPIs

- institution profiles completed
- data fields resolved
- workflows routed
- agent tasks completed
- evidence packages created
- approvals completed
- time to output
- exceptions detected
- opportunities detected
- opportunities activated
- value realized
- user adoption
- board reports generated

## Workflow KPIs

### Fintech Pilot

- time to readiness decision
- evidence completeness
- review cycle time
- pilot adoption
- target KPI improvement

### Exam

- request turnaround
- evidence completeness
- overdue items
- repeat findings
- remediation cycle time

### Participation

- package completion time
- matched buyers
- indication rate
- allocation time
- settlement exceptions

### Wealth / Insurance

- referrals
- consent rate
- conversion
- attributed revenue
- member retention

## Attribution

Value must distinguish:

- directly measured
- reasonably attributed
- estimated
- modeled
- unverified

## Executive Value Report

Include:

- baseline
- activity
- outcome
- evidence
- value
- risk reduction
- lessons
- scale recommendation

## Acceptance Tests

- Every claimed outcome has a source.
- Estimates are labeled.
- Attribution is not overstated.
- Baseline exists before pilot.
- Value report supports a scale decision.
