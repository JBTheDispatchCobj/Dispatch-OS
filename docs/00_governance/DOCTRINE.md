# Dispatch Doctrine

## Purpose
Dispatch increases the innovation capacity of institutions and the institutional capacity of innovation.

## Atomic concepts
1. **Object** — a durable entity, document, observation, event, claim, work item, or outcome.
2. **Profile** — the evolving operating context of an institution, company, person, product, or relationship.
3. **Relationship** — a first-class object connecting entities, with stage, evidence, history, visibility, opportunity, and next action.
4. **Intelligence Object** — one sourced unit of market meaning that can be rendered into many channels and lenses.
5. **Cartridge** — installable capability expressed through schemas, rules, prompts, workflows, metrics, permissions, and presentation contracts.
6. **Terminal** — an institution-specific operating environment assembled from profiles, feeds, products, workflows, and cartridges.

## Truth hierarchy
- Public fact
- Third-party claim
- Deterministic calculation
- Dispatch inference
- Institution-verified fact
- Private tenant fact
- Human-approved conclusion

Every consequential output must retain provenance, freshness, confidence, and visibility.

## Operating loop
Input → normalize → observe → interpret → propose → approve → execute → evidence → outcome → learn.

## Publication loop
Market event → Intelligence Object → role/institution/cartridge lens → channel variant → interaction → profile refinement → action.

## Commercial doctrine
- Free forever earns attention and builds market density.
- Low-friction paid adoption adds private context and persistence at a price an individual can absorb without committee approval.
- Usage and execution monetize actual work, integrations, pilots, diligence, administration, and expert escalation.
- No customer should need to buy a large fixed software contract merely to enter the network.
