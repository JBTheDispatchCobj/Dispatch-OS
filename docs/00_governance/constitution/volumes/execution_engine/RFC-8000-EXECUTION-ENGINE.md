\# Purpose

The Workflow Runtime is the execution engine that converts Execution Plans into observable operational work.

Execution Plans describe what should happen.

The Workflow Runtime makes it happen.

It coordinates humans, agents, connectors, approvals, timing, retries, and recovery while preserving complete operational context.

\---

\# Philosophy

Workflows are not automation scripts.

They are operational processes.

The Runtime is responsible for executing those processes safely, predictably, and transparently.

Execution should be deterministic.

Business logic should remain declarative.

\---

\# Core Principle

Execution Plans are static descriptions.

The Workflow Runtime is dynamic execution.

The Runtime never changes the plan.

It executes the current approved version of the plan.
